# Tóm tắt CD (2026-03-28)

Tài liệu này tóm tắt ngắn gọn trạng thái CD hiện tại của repo `AI-FOR-EDUCATION`.

## 1. Trạng thái hiện tại

- Repo đã có workflow CD tại `/.github/workflows/project-cd.yml`.
- CD hiện đã publish ảnh Docker lên GitHub Container Registry nhưng vẫn chưa triển khai lên server thật.
- Workflow CD tự chạy sau khi `CI Dự Án` trên nhánh `main` thành công.
- Vẫn có thể chạy tay bằng `workflow_dispatch`.

## 2. CD hiện đang làm gì

- Tạo metadata phát hành và ghi chú phát hành.
- Build frontend và tải artifact lên.
- Kiểm tra cú pháp backend rồi đóng gói mã nguồn backend.
- Resolve cấu hình `docker compose` rồi đóng gói bundle Docker.
- Build và publish ảnh Docker backend/frontend lên `ghcr.io`.
- Tạo báo cáo `deploy-placeholder` để xác nhận bản phát hành sẵn sàng.
- Gộp toàn bộ artifact thành `full-release-bundle-<sha>` để dễ tải về một lần.

## 3. Ảnh Docker phát hành

- Backend: `ghcr.io/<owner>/<repo>-backend`
- Frontend: `ghcr.io/<owner>/<repo>-frontend`
- Mỗi lần CD sẽ publish ít nhất 2 tag:
  - `placeholder-<sha7>`
  - `sha-<sha7>`
- Nếu chạy từ nhánh `main`, workflow sẽ publish thêm tag `latest`.

## 4. Tải artifact chính

- Mở GitHub Actions của workflow `CD Dự Án`.
- Chọn lần chạy CD gần nhất.
- Tải artifact `full-release-bundle-<sha>` ở phần Artifacts.

## 5. Khi CD thất bại

- Workflow sẽ tự tạo hoặc cập nhật GitHub Issue.
- Issue sẽ ghi rõ job nào lỗi và dẫn tới link GitHub Actions.

## 6. Ý nghĩa hiện tại

- Có khung CD rõ ràng dù chưa có server.
- Có thể kéo trực tiếp ảnh Docker từ `ghcr.io` để triển khai ở môi trường khác.
- Có artifact để kiểm tra, lưu trữ hoặc bàn giao.
- Khi có server thật, chỉ cần thay bước `deploy-placeholder` bằng bước triển khai thật.
